# DocCanvas Product Factory

Use this system for DocCanvas product surfaces that visualize Markdown-backed product knowledge as a living factory.

## Required sources

1. Read `README.md` for scope and provenance.
2. Import `tokens/colors_and_type.css` as the single token source.
3. Follow `brand/voice-and-tone.md` for user-facing copy.
4. Follow `brand/style-notes.md` for composition, depth, motion, and icon rules.

## Non-negotiable rules

- Preserve Markdown as the source of truth; presentation metadata never rewrites source content.
- Keep the building front-facing and readable. Use controlled CSS depth, not rotatable 3D.
- Use structural beams, floors, doors, and pipelines before introducing detached rounded cards.
- Use forest green for primary flow, oxidized copper for governance, and slate for engineering dependencies.
- Do not use purple AI gradients, emoji icons, decorative text glyphs, or colored left-border card strips.
- Digital employees are synthetic role representations. Do not imply a real person or live runtime state without source evidence.
- Respect `prefers-reduced-motion` and maintain WCAG AA text contrast.
